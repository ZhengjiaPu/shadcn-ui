## Introduction

The Input component provides a form input field or component that visually resembles an input field.

## Demo

This demo showcases the default Input component, representing a typical text field with basic styles.
It is intended for general-purpose text entry.

<ComponentPreview
  name="input-demo"
  className="[&_input]:max-w-xs"
  description="A form input component."
/>

## Variants

### Default Input

A standard input field allowing for text entry. This variant is used when a simple text field is needed
without any additional functionalities, enabling users to input plain text.

<ComponentPreview
  name="input-demo"
  className="[&_input]:max-w-xs"
  description="A form input component."
/>

### File Input

An input field for uploading files. This variant includes a file browser to allow users to select files from
their device. It is ideal for forms or applications that require file uploads, such as images or documents.

<ComponentPreview
  name="input-file"
  className="[&_input]:max-w-xs"
  description="A file input component."
/>

### Disabled Input

An input field that is disabled, preventing user interaction. This variant is used in scenarios where the input
is either optional or meant to be read-only under certain conditions.

<ComponentPreview
  name="input-disabled"
  className="[&_input]:max-w-xs"
  description="A disabled input component."
/>

### Input with Label

An input field with an accompanying label for context. This variant is often used in forms where labels are
necessary to inform the user of the expected content.

<ComponentPreview
  name="input-with-label"
  className="[&_input]:max-w-xs"
  description="An input component with a label."
/>

### Input with Button

An input field accompanied by a button for additional functionality. This variant is useful for scenarios where
user interaction is required alongside text input, such as searching or submitting a query.

<ComponentPreview
  name="input-with-button"
  className="[&_input]:max-w-xs"
  description="An input component with a button."
/>

### Input Form

A structured input component within a form. This example demonstrates an input that is part of a larger form,
integrated with other components to collect multiple fields of information.

<ComponentPreview name="input-form" />

### Input with Number

An input field specifically designed for numeric entry. This variant restricts the input to numbers, making it
ideal for forms that require numerical values, such as age or quantity.

<ComponentPreview
  name="input-with-number"
  className="[&_input]:max-w-xs"
  description="An input component for entering numbers."
/>

### Input with Phone Number

An input field configured for phone number entry based on selected country. This variant provides validation
and formatting for phone numbers, helping users enter their phone numbers correctly.

<ComponentPreview
  name="input-with-phonenumber"
  className="[&_input]:max-w-xs"
  description="An input component for entering phone numbers based on selected country."
/>
