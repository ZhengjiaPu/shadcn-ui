## Introduction

The Breadcrumb component is a navigational aid that displays the current page’s location within a hierarchical structure, allowing users to trace their path back to previous pages. Commonly used in multi-level or deeply nested interfaces, Breadcrumbs improve user experience by providing context, enhancing navigation, and reducing the effort required to move through the application. This component supports customization options such as separators and responsive dropdowns, making it a flexible solution for various design requirements.

## Variants

### breadcrumb with custom separator

This variant allows you to use a custom separator between breadcrumb items. Instead of the default “/” or “>” symbol, you can specify any icon or character as a separator by passing a custom component to the BreadcrumbSeparator. This variant is useful when the design calls for unique symbols that align with the application's aesthetic, making the navigation experience more cohesive.

<ComponentPreview
  name="breadcrumb-separator"
  description="A breadcrumb with a custom separator"
/>

```tsx showLineNumbers {1,10-12}
import { Slash } from "lucide-react"

...

<Breadcrumb>
  <BreadcrumbList>
    <BreadcrumbItem>
      <BreadcrumbLink href="/">Home</BreadcrumbLink>
    </BreadcrumbItem>
    <BreadcrumbSeparator>
      <Slash />
    </BreadcrumbSeparator>
    <BreadcrumbItem>
      <BreadcrumbLink href="/components">Components</BreadcrumbLink>
    </BreadcrumbItem>
  </BreadcrumbList>
</Breadcrumb>
```

### breadcrumb with dropdown

The Breadcrumb with Dropdown variant is ideal for navigating complex, multi-level hierarchies. Instead of showing all breadcrumb items in a linear path, this variant groups sections into a dropdown. Users can expand the dropdown to view or select intermediate levels, providing a cleaner, more concise navigation experience. This variant is particularly useful in applications where the breadcrumb path can be long, reducing visual clutter while retaining full navigational functionality.

### breadcrumb with collapsed

The Collapsible Breadcrumb variant is designed for interfaces with limited space or a large number of hierarchical levels. In this variant, only the first and last items are displayed by default, while the intermediate items are collapsed into a dropdown. This approach keeps the navigation compact and accessible, especially on smaller screens, while still allowing users to trace their path through the hierarchy.

<ComponentPreview
  name="breadcrumb-ellipsis"

  description="A breadcrumb showing a collapsed state."
/>

```tsx showLineNumbers {1,9}
import { BreadcrumbEllipsis } from "@/components/ui/breadcrumb"

...

<Breadcrumb>
  <BreadcrumbList>
    {/* ... */}
    <BreadcrumbItem>
      <BreadcrumbEllipsis />
    </BreadcrumbItem>
    {/* ... */}
  </BreadcrumbList>
</Breadcrumb>
```

### breadcrumb with link component

The Breadcrumb with Link Component variant integrates interactive links for each breadcrumb item, enabling users to click and navigate directly to any previous page in the hierarchy. This approach enhances usability by allowing immediate access to any level, making it a preferred choice for dynamic applications where users frequently navigate between different levels. This variant is especially helpful in content-heavy applications, like e-commerce or knowledge bases, where users may need to backtrack quickly.

<ComponentPreview
  name="breadcrumb-link"
  description="A breadcrumb with a custom Link component"
/>

```tsx showLineNumbers {1,8-10}
import { Link } from "next/link"

...

<Breadcrumb>
  <BreadcrumbList>
    <BreadcrumbItem>
      <BreadcrumbLink asChild>
        <Link href="/">Home</Link>
      </BreadcrumbLink>
    </BreadcrumbItem>
    {/* ... */}
  </BreadcrumbList>
</Breadcrumb>
```

---

### breadcrumb with responsive

The Responsive Breadcrumb variant is optimized for devices of varying screen sizes, ensuring seamless navigation across desktops, tablets, and mobile devices. This variant automatically adjusts the display of breadcrumb items based on available screen width, using techniques like collapsing items into a dropdown on smaller screens. It allows users to maintain context regardless of the device they are using, improving accessibility and adaptability in multi-device applications.

## Usage

```tsx
import {
  Breadcrumb,
  BreadcrumbItem,
  BreadcrumbLink,
  BreadcrumbList,
  BreadcrumbPage,
  BreadcrumbSeparator,
} from "@/components/ui/breadcrumb"
```

```tsx
<Breadcrumb>
  <BreadcrumbList>
    <BreadcrumbItem>
      <BreadcrumbLink href="/">Home</BreadcrumbLink>
    </BreadcrumbItem>
    <BreadcrumbSeparator />
    <BreadcrumbItem>
      <BreadcrumbLink href="/components">Components</BreadcrumbLink>
    </BreadcrumbItem>
    <BreadcrumbSeparator />
    <BreadcrumbItem>
      <BreadcrumbPage>Breadcrumb</BreadcrumbPage>
    </BreadcrumbItem>
  </BreadcrumbList>
</Breadcrumb>
```
