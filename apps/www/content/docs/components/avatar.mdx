## Introduction

The Avatar component represents a user visually, displaying an image when available and a fallback (such as initials or a default icon) when it is not. This component is essential for user identification and personalization in interfaces.

## Demo

/>

The demo highlights the core functionality of the Avatar component. In its standard use, the Avatar component will attempt to display an image sourced from the user's profile or specified link. If the image fails to load or is not provided, the component defaults to a fallback. This fallback can be customized to show the user's initials or a default icon, ensuring a consistent visual presence for each user even in the absence of an image.

The avatar component is highly flexible and responsive, allowing developers to define various fallback mechanisms based on context. This ensures that the user interface maintains a uniform look while still being informative and recognizable to users. ## Usage

```tsx
import { Avatar, AvatarFallback, AvatarImage } from "@/components/ui/avatar"
```

```tsx
<Avatar>
  <AvatarImage src="https://github.com/shadcn.png" />
  <AvatarFallback>CN</AvatarFallback>
</Avatar>
```
