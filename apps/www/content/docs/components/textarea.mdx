The Textarea component provides an interactive multi-line text input field, designed for capturing longer textual data. Whether used in forms, comments sections, or notes, this component ensures a responsive and accessible user experience. This component is perfect for scenarios such as comment sections, note-taking apps, and feedback forms. It ensures usability and accessibility while allowing developers to customize its appearance and behavior.

## Examples

### Default

The DefaultTextarea variant represents the base implementation of the textarea component. It provides a clean and straightforward multi-line text input field, suitable for capturing user inputs such as notes, messages, or descriptions.

<ComponentPreview name="textarea-demo" description="A simple textarea demo." />

### Disabled

The DisabledTextarea variant is a non-interactive version of the textarea component. It visually communicates its disabled state using reduced opacity and a "not-allowed" cursor. This is ideal for read-only or unavailable input scenarios.

<ComponentPreview
  name="textarea-disabled"
  description="A disabled textarea demo."
/>

### With Label

The TextareaWithLabel variant includes an accessible label paired with the textarea. This design ensures that users have clear guidance when filling out the input, especially in form-heavy interfaces.

<ComponentPreview
  name="textarea-with-label"

### With Text

The TextareaWithText variant includes placeholder or helper text inside the textarea, guiding users on the expected input format or content. This is ideal for providing context without requiring additional labels.

### With Button

The TextareaWithButton variant integrates a button next to the textarea for additional functionality, such as submitting or clearing input. This is commonly used in chat applications or feedback forms.
