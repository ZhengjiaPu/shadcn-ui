## Introduction

The `NavigationMenu` component is a streamlined and efficient navigation tool, suitable for building a collection of links within a website. This component provides flexible configuration options to embed navigation menus in various parts of a page, allowing users to quickly browse website content. Supporting a multi-level link structure, it enhances user experience with smooth navigation. The NavigationMenu component follows the WAI-ARIA design guidelines, ensuring accessibility for all users.

## Demo

The `NavigationMenu` demo demonstrates a basic implementation of a navigational structure, where multiple items can be grouped within a list for structured, hierarchical navigation. Each `NavigationMenuItem` contains a `NavigationMenuTrigger` and `NavigationMenuContent`, allowing users to access links and nested menu options in an organized way.

In this example, clicking on a trigger reveals the associated content, which can include links, categories, or additional navigation layers, providing a flexible, multi-level navigation system. This setup is ideal for websites with diverse content sections, as it enables users to explore related content quickly and intuitively. The layout ensures that navigation elements remain visually accessible and responsive across different devices.

## Usage Example

```tsx
import {
  NavigationMenu,
  NavigationMenuContent,
  NavigationMenuIndicator,
  NavigationMenuItem,
  NavigationMenuLink,
  NavigationMenuList,
  NavigationMenuTrigger,
  NavigationMenuViewport,
} from "@/components/ui/navigation-menu"
```

```tsx
<NavigationMenu>
  <NavigationMenuList>
    <NavigationMenuItem>
      <NavigationMenuTrigger>Item One</NavigationMenuTrigger>
      <NavigationMenuContent>
        <NavigationMenuLink>Link</NavigationMenuLink>
      </NavigationMenuContent>
    </NavigationMenuItem>
  </NavigationMenuList>
</NavigationMenu>
```

## Examples

### Link Component

When using the Next.js `<Link />` component, you can use `navigationMenuTriggerStyle()` to apply the correct styles to the trigger.

```tsx
import { navigationMenuTriggerStyle } from "@/components/ui/navigation-menu"
```

```tsx {3-5}
<NavigationMenuItem>
  <Link href="/docs" legacyBehavior passHref>
    <NavigationMenuLink className={navigationMenuTriggerStyle()}>
      Documentation
    </NavigationMenuLink>
  </Link>
</NavigationMenuItem>
```

See also the [Radix UI documentation](https://www.radix-ui.com/docs/primitives/components/navigation-menu#with-client-side-routing) for handling client-side routing.
