## Introduction

The `Input OTP` component provides an accessible interface for entering one-time passwords (OTP), typically used for authentication. It includes support for numeric and alphanumeric patterns and offers customization for separators between digit groups. Built with accessibility in mind, `Input OTP` follows ARIA guidelines to ensure compatibility with assistive technologies. This component is suitable for secure login interfaces, verification steps, and similar applications that require user-friendly OTP entry.

## Variants

### Pattern

The `Pattern` variant allows customization of acceptable input formats. You can set it to accept only numeric values or extend it to include alphanumeric characters. This is helpful for applications that require flexible OTP structures, enhancing user experience by clearly defining acceptable inputs.

<ComponentPreview
  name="input-otp-pattern"
  description="An OTP input field configured with an alphanumeric pattern."
/>

```tsx showLineNumbers {1,7}
import { REGEXP_ONLY_DIGITS_AND_CHARS } from "input-otp"

...

<InputOTP
  maxLength={6}
  pattern={REGEXP_ONLY_DIGITS_AND_CHARS}
>
  <InputOTPGroup>
    <InputOTPSlot index={0} />
    {/* ... */}
  </InputOTPGroup>
</InputOTP>
```

### Separator

The `Separator` variant uses a visual separator between digit groups, ideal for enhancing readability in longer OTPs. Separators allow for easy grouping of digits, making it user-friendly for input scenarios where OTPs are lengthy or structured.

<ComponentPreview
  name="input-otp-separator"
  description="An OTP input with customizable separators between groups."
/>

```tsx showLineNumbers {4,15}
import {
  InputOTP,
  InputOTPGroup,
  InputOTPSeparator,
  InputOTPSlot,
} from "@/components/ui/input-otp"

...

<InputOTP maxLength={6}>
  <InputOTPGroup>
    <InputOTPSlot index={0} />
    <InputOTPSlot index={1} />
  </InputOTPGroup>
  <InputOTPSeparator />
  <InputOTPGroup>
    <InputOTPSlot index={2} />
    <InputOTPSlot index={3} />
  </InputOTPGroup>
</InputOTP>
```

### Controlled

The `Controlled` variant enables external control over the OTP input value, allowing synchronization with state management or other form components. This variant is useful in complex forms where OTP input needs to be managed centrally.

## Usage

```tsx
import {
  InputOTP,
  InputOTPGroup,
  InputOTPSeparator,
  InputOTPSlot,
} from "@/components/ui/input-otp"
```

```tsx
<InputOTP maxLength={6}>
  <InputOTPGroup>
    <InputOTPSlot index={0} />
    <InputOTPSlot index={1} />
    <InputOTPSlot index={2} />
  </InputOTPGroup>
  <InputOTPSeparator />
  <InputOTPGroup>
    <InputOTPSlot index={3} />
    <InputOTPSlot index={4} />
    <InputOTPSlot index={5} />
  </InputOTPGroup>
</InputOTP>
```
